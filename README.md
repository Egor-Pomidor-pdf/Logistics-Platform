# Logistics Platform

Платформа отслеживания отправлений. Принимает события об изменении
состояния отправлений от партнёров и внутренних систем, сохраняет историю,
предоставляет API и готовит данные для аналитики и прогнозирования сроков доставки.

---

## Архитектура

```
[Партнёры / внутренние системы]
         │
         ▼ HTTP или Kafka
 logistics-normalizer-service
         │
         ▼ Kafka
  shipment-core-service  ──▶  REST / gRPC API
         │
         ├──────────────────────────┐
         ▼                          ▼
 eta-data-service          analytics-exporter
         │                          │
         ▼ HTTP                     ▼
  ETA / SLA сервисы         ClickHouse (аналитика)
```

---

## Сервисы

### logistics-normalizer-service

Точка входа для событий от партнёров и внутренних систем.
Принимает сырые события по HTTP или из Kafka, приводит их к единому формату:
нормализует коды статусов, прогнозируемые сроки доставки и временны́е зоны,
отфильтровывает дубликаты. Валидные события публикует дальше в Kafka;
невалидные — в отдельный поток для разбора.

**Стек:** Go · Kafka · Redis · Prometheus · OpenTelemetry

---

### shipment-core-service

Центральный сервис платформы — система записи состояния отправлений.
Потребляет нормализованные события из Kafka, сохраняет иммутабельную историю
и актуальный статус каждого отправления. Предоставляет REST и gRPC API
для чтения данных внутренними и внешними клиентами.
Доступ к API защищён ролевой авторизацией (JWT).

**Стек:** Go · PostgreSQL · Kafka · REST (chi) · gRPC · Prometheus · OpenTelemetry

---

### eta-data-service

Сервис подготовки данных для моделей прогнозирования сроков доставки (ETA) и SLA-контроля.
Читает историю отправлений, строит агрегированные снимки данных —
длительности статусов, возраст, количество событий — и отдаёт их внешним ML-сервисам
по запросу или пакетами. Результаты кэшируются; при промахе кэша пересчёт
выполняется асинхронно в фоне.

**Стек:** Go · PostgreSQL · Redis · RabbitMQ · REST (chi) · Prometheus · OpenTelemetry

---

### analytics-exporter

Сервис инкрементальной выгрузки данных для аналитики и SLA-отчётности.
Читает историю и состояния отправлений из PostgreSQL и переносит их в ClickHouse
батчами, отслеживая позицию последнего экспорта.
Безопасно работает в нескольких репликах.

**Стек:** Go · PostgreSQL · ClickHouse · Prometheus · OpenTelemetry

---

## Инфраструктурные зависимости

- **PostgreSQL** — основное хранилище событий и состояний
- **Kafka** — транспорт между normalizer и core
- **Redis** — дедупликация входящих событий, кэш агрегатов
- **RabbitMQ** — асинхронный пересчёт данных в eta-data-service
- **ClickHouse** — аналитическое хранилище для SLA-отчётов
